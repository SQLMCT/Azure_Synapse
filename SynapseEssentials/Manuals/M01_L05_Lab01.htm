Create, Query and Login to Azure Synapse SQL

Introduction

This lab introduces students to Azure Synapse SQL Pool. After this lab you should have a clear understanding on how to create and provision Synapse SQL Pool and how to connect to Azure Synapse SQL Pool and create a SQL login.

Estimated Time

90 minutes

Objectives

At the end of this lab, you will be able to:

·       Create a Resource Group

·       Provision an Azure Data Lake Gen2 Storage Account

·       Provision an Azure Synapse Workspace

·       Provision an Azure Synapse SQL Pool from Azure Synapse Studio

·       Enabling Transparent Data Encryption

·       Query Synapse SQL from SQL Server Management Studio

·       Create SQL Login

·       Upload datasets to Azure Data Lake as staging files for loading into Synapse SQL Pool

 

 

 

Prerequisites

·       An Azure subscription. See Get Azure free trial.

·       Supported operating system to perform this class: Windows 10 / Windows Server 2016

·       Download and install Visual Studio with option “Data Storage and processing”

·       Download and install SQL Server Management Studio.

https://docs.microsoft.com/en-us/sql/ssms/download-sql-server-management-studio-ssms?view=sql-server-ver15

 

 

Note: The labs can be performed using your laptop environment rather than LOD environment, except for Exercise 9 – that steps needs to be completed from LOD VM.

Exercise 8: Creating Logins

To add SQL Server Authenticated user to Synapse SQL we need to perform the following steps:

·         Create new server login

·         Add user to master database using this new server login

·         Add user to Synapse SQL using this new server login

 

Tasks

Add user in master database

1.        With the connection opened to Synapse SQL, right click the server name and click New Query.

2.        Ensure master database is selected for the session of the query that opened.

3.        Type in the below statement to create a new server login by executing below statement in query window.

CREATE LOGIN DWLoader WITH PASSWORD = 'Str0ng_password';

CREATE LOGIN dwloaderrc10 WITH PASSWORD = 'Str0ng_password';

 

4.        Type in the below statement to create a user in the master database for the login created above.

CREATE USER DWLoader FOR LOGIN DWLoader;

CREATE USER dwloaderrc10 FOR LOGIN dwloaderrc10;

 

 

5.        Type in the name of your Synapse SQL Pool (might be different from the one shown below) in the query database field as shown below:

 

 

6.        To create user in Synapse SQL using login created in earlier steps, execute below statement.

CREATE USER DWLoader FOR LOGIN DWLoader;

CREATE USER dwloaderrc10 FOR LOGIN dwloaderrc10;

Exercise 9: Uploading Dataset to Azure Data Lake

In this exercise, datasets will be uploaded to the Azure Data Lake that was created in Exercise 2. This dataset will be used in the next upcoming modules. The dataset New York Taxi trips will be used in this workshop. The dataset is publicly made available by the NYC Taxi and Limousine Commission. For this workshop, one year of Yellow taxi trip data for the year 2018 will be loaded. This workshop covers the approach on loading Parquet files and CSV files. This exercise would take 20 minutes to upload the data to Azure Data Lake.

 

Tasks

Upload datasets to Azure Data Lake

1.        Navigate to https://portal.azure.com. In the list of resources, select the Synapse Workspace which was provisioned in Exercise 3.

2.        In the Synapse Workspace page within the Azure Portal, notice the Primary ADLS Gen2 account URL.

3.        Noting down the storage account name (the text before the .dfs statement in the URL) in Step (2) above, open the Storage Account resource in the Azure Portal. This Storage Account should be the same as created in Exercise 3.

4.        In the Storage Account pane, scroll down to Data Lake Storage section. Click on Containers.

 

5.        Click + Container on the top to create a new container. Name the container as nyctaxistaging and click Create.

6.        Within the container created, click + Add Directory to create a new folder. Name the directory as fact. Repeat this step to create another container named dimension.

 

 

 

7.        Click on the fact directory. Click on Upload button at the top. Locate the files from Labs\Files\Module 1\Dataset\Fact directory and upload all the files. Depending on the network speed, you may want to upload in batches of 2-3 files.

8.        Click on the container nyctaxistaging to navigate back to the root folder.

9.        Click on the dimension directory. Click on Upload button at the top. Locate the files from Labs\Files\Module 1\Dataset\Dimension directory and upload all the files.

Exercise 10: Take Note of Storage Account Keys

Tasks

Copy Storage Account access key

2.        Within the Storage Account page on the left pane, click Access keys.

 

3.        Take note of key 1. You can copy and paste the key to a Notepad file.

 

4.        Next from the left pane, click on Shared access signature. This is to obtain the SAS key.

5.        Within the Shared access signature pane, select Service, Container and Object under Allowed resource types.

6.        Click Generate SAS and connection string. Copy and paste the SAS token populated to a Notepad file.
